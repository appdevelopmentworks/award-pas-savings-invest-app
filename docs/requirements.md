# 要求定義（MVP）
作成日: 2026-01-06

## 1. 目的
投資未経験〜初心者の会社員が「積立投資を始めたいが、いくら積み立ててよいか分からない」問題を解決する。
家計を壊さずに継続できる **“安全な積立額”** と **“今月の最優先アクション”** を自動提示する。

## 2. ターゲット
- 20代後半〜40代の会社員
- 投資未経験〜初心者（これから積立開始）
- 家計簿は続かない/記録が面倒
- NISA等に興味はあるが「金額設計」ができない

## 3. 成功指標（MVPのKPI）
- 初回入力完了率（フォーム完了）: 60%+
- 推奨プラン閲覧率: 80%+
- 「今月のToDo」保存（チェック）率: 30%+
- 7日後の再訪: 15%+

## 4. コアユーザーストーリー
### US-1: はじめての積立額を知りたい
ユーザーは手取り・固定費・変動費・貯金などを入力し、今月から始める「安全な積立額」を知る。

### US-2: 不安（突発出費）に備えたい
ユーザーは生活防衛資金の不足を理解し、積立投資より先に確保すべき金額を知る。

### US-3: 続く仕組みを作りたい
ユーザーは「今月の最優先ToDo」および「3ステップ計画」を得て、次の行動が明確になる。

## 5. 機能要件（MVP）
### 5.1 入力（5分で終わる）
- 月の手取り（after-tax）
- 固定費（月額合計）
- 変動費（月額合計）
- 現金残高（貯金）
- 特別費（年額見込み or テンプレ選択→推定）
- 借入状況（なし/あり：毎月返済額）
- 家族構成（大人/子ども）

**任意**
- 目標（複数可：○年後に○円、もしくは目的選択）
- 会社員のボーナス有無（年2回 / 金額レンジ）
- ライフイベント（住宅/教育など）

### 5.2 出力
- 積立可能額レンジ（2段階）
  - 安全ライン（絶対続く）
  - 成長ライン（慣れたら増やす）
- 3ステップ計画（守る→小さく始める→段階増額）
- 今月の最優先ToDo（1つ）＋補助ToDo（最大3つ）
- 目標がある場合：達成シミュレーション（ざっくりでOK）

### 5.3 保存
- localStorageに保存
  - 入力値
  - 推奨結果
  - ToDoのチェック状態
  - 直近の計算日時
  - リマインド設定

### 5.4 ナビゲーション（最小）
- / … LP（価値説明、開始CTA）
- /onboarding … 入力フォーム（ステップ式）
- /result … 結果（カード＋チャート＋ToDo）
- /settings … データ削除、免責、言語（将来）

## 6. 非機能要件
- 表示：スマホ優先（レスポンシブ）
- 入力は極力少なく（単位/例を明確に）
- 速度：結果計算はクライアントで即時
- 信頼性：計算根拠がわかる（“なぜこの金額？”説明）
- アクセシビリティ：ラベル/エラー表示を丁寧に

## 7. 制約
- 外部口座連携なし
- 金融商品推奨はしない（一般的な説明のみ）
- 利用規約/免責を明示

## 8. 受け入れ基準（サンプル）
- 必須項目が妥当な範囲で入力されると、結果ページで「安全ライン/成長ライン/ToDo」が必ず表示される
- localStorageを消してもクラッシュしない（初回状態に戻る）
- 入力が欠ける/異常値のとき、わかりやすいエラーが表示される
